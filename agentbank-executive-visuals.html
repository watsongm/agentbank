When agents are the interface Core concepts · five exhibits
Exhibit 1 · The stack

When a person's primary tool is an agent, every service sits in the same four-layer stack.

People stop opening apps; they tell an agent what they want. The agent reaches into domain rails — finance, commerce, travel, health — through typed, authenticated interfaces. Underneath everything is a shared trust fabric.

Core idea
The person-facing layer is the agent. Everything else — including the bank — is a rail behind it, plugged in through the same small set of trust primitives.
Exhibit 2 · Features become primitives

Products stop being screens. They become small, named capabilities an agent can call.

A modern bank app has hundreds of screens. An agent needs none of them. What it needs is a handful of well-named tools — and the freedom to compose them into whatever the user actually asked for.

Exhibit 3 · User experience

The person says one sentence. The agent does the rest — visibly.

Every step is typed, scoped, and logged. The person never opens an app, fills a form, or copies a reference number. Trust comes from plainly-stated scope and a replayable record, not from a login screen.

01
Ask
“I got paid — move anything spare into my house fund if it's safe.”
Plain language

One prompt: read balances, move up to £500, expires in 24 hours. One tap to approve.

03

Agent plans, calls the right tools in order, and explains its reasoning as it goes.
Tool calls are visible
04
Record
Every step is written to an immutable log. The person can replay it; the regulator can inspect it.
Audit trail
05
Report
“Moved £420 to your house fund. Want me to do this every payday?”
No app opened
What it feels like
A short conversation, one permission prompt, a confirmation. No login, no form, no dashboard.
What actually happened
A handful of typed API calls on a scoped, time-bound token — with a full reasoning trail attached. More inspectable than any screen-based journey.
Core idea
Trust is not the login page. It's clear scope + visible reasoning + replayable record.
Exhibit 4 · The lifestyle agent

Most consumers end up with one agent across most of life. Every domain plugs into it.

The lifestyle agent is the interface. Around it: the domain rails the person relies on — finance, commerce, travel, health, utilities — all reached through the same kind of typed, consented tool calls.

01 · The channel collapses
Apps stop being where life happens.
They persist for inspection and edge cases, but day-to-day action moves into conversation with the agent.
02 · Products become tools
The unit of value shifts from screens to callable capabilities.
If a capability isn't exposed as a clean tool, an agent can't reach it — and the customer never sees it.
03 · Consent replaces logins
Permission becomes the new interface primitive.
Scope, expiry, and bound-to-agent rules matter more than passwords and sessions.
04 · Audit replaces support
Disputes are resolved by replaying reasoning trails.
What the agent saw, planned, and did is the source of truth — for the user, the provider, and the regulator.
05 · Distribution changes
Whoever agents prefer to call wins the relationship.
Providers compete on tool quality, reliability, and trust — not on app downloads or brand placement.
Core idea
One agent, many rails. The consumer's relationship is with the agent; each provider's relationship is with the tool calls the agent chooses to make.
Exhibit 5 · Blended outcomes

The most useful outcomes come from blending tools across ecosystems, not from any single rail.

When one agent can see your bank, your inbox, your calendar, your shopping and your messages, it stops being a better banking app and starts being life admin that just … happens. These are outcomes no single provider can deliver alone.

Core idea
Any one ecosystem — bank, email, travel — only sees a slice of the problem. The agent's superpower is that it stands above all of them, composing their tools into outcomes the user actually cares about. The more ecosystems the agent can reach, the bigger the outcomes become — and the more the agent, not any single provider, becomes the place life gets organised.
